# Project Structure

```

│   │   ├── +page.svelte          # Route pages

│   │   │   └── auth/            # Auth-related components
│   │   ├── stores/              # Svelte stores (using runes)
│   │   ├── assets/              # Static assets (images, icons)
│   │   ├── server/              # Server-only code
│   │   │   ├── auth/           # Better Auth configuration

│   │   ├── auth-client.ts      # Client-side auth utilities
│   │   └── index.ts            # Public library exports
│   ├── app.html                 # HTML template
│   ├── app.css                  # Global styles

├── static/                       # Static files served at root
├── drizzle/                      # Database migrations
└── [config files]               # Root-level configuration
```

## Key Conventions

### Remote Functions Architecture

**Critical pattern**: All server interactions use remote functions, not load functions or form actions.

- Define server functions in `*.remote.ts` files alongside routes
- Import types: `query`, `form`, `command` from `$app/server`
- Components import and call these functions directly
- Example:
  ```typescript
  // routes/data.remote.ts
  import { query } from '$app/server';
  export const getData = query(async () => { ... });
  
  // routes/+page.svelte
  import { getData } from './data.remote';
  const data = await getData();
  ```

### Database Schema

- Main schema: `src/lib/server/db/schema.ts`
- Auth schema: `src/lib/server/db/auth-schema.ts` (generated by Better Auth)
- Use Drizzle ORM patterns with PostgreSQL dialect
- Export TypeScript types from schema: `typeof table.$inferSelect`
- Use enums for constrained values (e.g., `dimensionEnum`)

### Component Organization

- Components in `src/lib/components/`
- Group by feature (e.g., `auth/`, `quiz/`, `dashboard/`)
- Use Svelte 5 runes (`$state`, `$derived`, `$effect`)
- Style with Tailwind CSS classes
- Use Bits UI for headless primitives

### Server Code

- All server-only code in `src/lib/server/`
- Never import server code directly in client components
- Use remote functions for client-server communication
- Database client exported from `src/lib/server/db/index.ts`

### Styling Conventions

- Tailwind CSS for all styling
- Custom playful design (not default component themes)
- Mobile-first responsive design
- Use Bits UI for accessible component primitives
- Tabler Icons for iconography

### TypeScript

- Strict mode enabled
- Path alias `$lib` for `src/lib`
- Export types from schema files
- Use type inference where possible

### File Naming

- Routes: `+page.svelte`, `+layout.svelte`
- Remote functions: `*.remote.ts`
- Components: PascalCase (e.g., `LoginForm.svelte`)
- Utilities: camelCase (e.g., `auth-client.ts`)
- Types: Exported from schema or defined inline

## Database Tables

Core tables for personality tracking:

- `questions` - MBTI assessment questions
- `responses` - User answers to questions
- `user_stats` - Streak tracking and onboarding status
- `personality_snapshots` - Historical personality calculations
- `user`, `session`, `account`, `verification` - Better Auth tables

## Configuration Files

- `svelte.config.js` - SvelteKit config (enables remote functions)
- `vite.config.ts` - Vite build configuration
- `drizzle.config.ts` - Database configuration
- `tsconfig.json` - TypeScript configuration
- `eslint.config.js` - Linting rules
- `.prettierrc` - Code formatting (tabs, single quotes, 100 width)
