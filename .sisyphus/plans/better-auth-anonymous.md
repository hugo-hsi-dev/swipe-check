### Research Findings

**Better Auth SvelteKit Hook Setup** (from better-auth.com/docs/integrations/svelte-kit):

```typescript
// src/hooks.server.ts
import { auth } from '$lib/server/auth';
import { svelteKitHandler } from 'better-auth/svelte-kit';
import { building } from '$app/environment';

export async function handle({ event, resolve }) {
	// Fetch current session from Better Auth
	const session = await auth.api.getSession({
		headers: event.request.headers
	});

	// Make session and user available on server
	if (session) {
		event.locals.session = session.session;
		event.locals.user = session.user;
	}

	return svelteKitHandler({ event, resolve, auth, building });
}
```

**Key Points**:

- `svelteKitHandler` processes all auth routes automatically
- Populate `event.locals.session` and `event.locals.user` for server-side access
- Import `building` from `$app/environment` to handle build-time properly

**Anonymous Auth Usage**:

```typescript
// Client-side
import { authClient } from 'better-auth/svelte';
const session = authClient.useSession();

// Sign in anonymously
await authClient.signIn.anonymous();

// Sign out
await authClient.signOut();
```

### Test Decision

- **Infrastructure exists**: YES (Vitest + Playwright configured)

- **QA approach**: TDD - write failing test first, then implement

### TDD Methodology (RED-GREEN-REFACTOR)

**Every feature MUST follow this cycle:**

1. **RED Phase** - Write the test:
   - Write test that expects the feature to work
   - Run test: should FAIL (red)
   - Commit: `test(auth): add test for [feature] - RED`

2. **GREEN Phase** - Make it pass:
   - Write MINIMUM code to make test pass
   - Run test: should PASS (green)
   - No refactoring yet!
   - Commit: `feat(auth): implement [feature] - GREEN`

3. **REFACTOR Phase** - Clean up:
   - Refactor code while keeping tests green
   - Run test: should still PASS
   - Commit: `refactor(auth): clean up [feature] - REFACTOR`

**Task Structure**:
Each task includes 3 sub-tasks:

- `[Feature] - RED`: Write failing test
- `[Feature] - GREEN`: Implement to pass test
- `[Feature] - REFACTOR`: Clean up implementation

## Feature Branch Workflow

**Branch Name**: `feature/better-auth-anonymous`

**TDD Workflow with Git**:

1. Create feature branch from main: `git checkout -b feature/better-auth-anonymous`
2. For each feature:
   - **RED Phase**: Write test, commit with "- RED" suffix
   - **GREEN Phase**: Implement, commit with "- GREEN" suffix
   - **REFACTOR Phase**: Clean up, commit with "- REFACTOR" suffix
3. Push branch regularly: `git push -u origin feature/better-auth-anonymous`
4. After all tasks complete, verify everything works
5. Create PR for review
6. Merge after approval

**Why TDD Commits Matter**:

- Shows the evolution of the code
- Makes it easy to review the thought process
- Allows reverting to any phase if needed
- Documents the RED-GREEN-REFACTOR cycle
